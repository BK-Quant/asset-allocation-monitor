# CLAUDE.md — 자산배분모니터 프로젝트

## 프로젝트 개요
매달 주요 정적/동적 자산배분 전략의 "이번달 보유해야 할 티커·비중"을 전월 말 종가 기준으로 계산해 보여주는 무료 공개 웹사이트.

참고 사이트: https://jasan-calc.netlify.app/ (전략 로직·데이터 소스 구조를 그대로 참고, UI/UX는 독자적으로 새로 디자인. 일부 전략은 추후 브라이언 자체 전략으로 교체 예정)

## 확정된 사양

### 전략 (18개, 참고 사이트와 동일하게 시작)
- 한국 ETF 정적배분 3종 (안정형/중립형/성장형)
- 영구포트폴리오(PERM)
- 듀얼모멘텀 3종: 전통듀얼모멘텀(DUAL), 종합듀얼모멘텀(CDM), 가속듀얼모멘텀(ADM)
- LAA, RAA, GTAA, PAA, VAA, FAA, AAA, DAA, DGA, 채권동적배분(DYNBOND)
- 계산 로직은 참고 사이트 `js/engine.js`를 그대로 이식 (모멘텀 스코어, SMA, 최소분산 등)
- **일부 전략은 추후 브라이언 자체 전략으로 교체 예정** (교체 대상 미정 — 슬롯 구조로 설계해 교체 쉽게)

### 데이터 소스
- **가격**: Yahoo Finance 공개 차트 API (`query1.finance.yahoo.com/v8/finance/chart/{ticker}`), 키 불필요. 참고 사이트와 동일 소스 사용 ("가장 많이 쓰는 플랫폼")
- **매크로**: BLS(실업률), FRED(T10Y3M 스프레드), us500.com(S&P500 배당수익률) — DGA/RAA/GTAA 등 일부 전략에 필요
- ⚠️ **참고 사이트와의 차이**: 참고 사이트는 최근 252거래일만 저장하고 날짜를 노출하지 않음(배열 인덱스만 존재). 이 프로젝트는 **백테스트 그래프**를 위해 훨씬 긴 히스토리(수년치)를 날짜와 함께 저장해야 함 → 데이터 스키마를 `{"dates": [...], "SPY": [...], ...}` 형태로 변경 (날짜-가격 인덱스 매칭 필요)

### 계산 시점 원칙 (중요, 참고 사이트와의 핵심 차이)
- 참고 사이트는 "최신 종가"로 매일 신호가 바뀜. 이 프로젝트는 **"전월 말 종가" 기준으로 고정**해서 "이번달 보유 티커"를 계산 (월 1회 확정, 월중 흔들리지 않음)
- 모든 전략 패널에 **타이밍 배지** 표시: "기준: YYYY-MM-DD 종가(전월 말) → 적용: YYYY년 M월 보유"
- 예외: 가속듀얼모멘텀(ADM)은 위 확정치 외에 **"차월 예상 티커"**를 별도로 추가 표시 (최신 가용 데이터로 계산한 프리뷰, 월말 확정 전까지 바뀔 수 있음을 명시)

### 커스텀 지표 섹션 (별도 표시 요청)
전략 목록과 별개로 "커스텀 지표" 섹션을 두고 아래 3가지를 모아서 보여줌:
1. DAA 전략의 카나리아 자산 2종(VWO, BND)의 **원시 가중 모멘텀 스코어 값** (risk-on/off 판정 결과가 아니라 실제 수치)
2. 가속듀얼모멘텀(ADM)의 **당월 확정 티커**
3. 가속듀얼모멘텀(ADM)의 **차월 예상 티커** (프리뷰, 미확정 표시)

### 추가 시각화
- **티커별 주가 그래프**: 각 전략에서 사용되는 티커의 과거 가격 추이 차트
- **전략별 백테스트 그래프**: 각 전략을 과거 데이터에 월별 리밸런싱으로 소급 적용했을 때의 누적 성과(NAV) 곡선. 프론트엔드 부하를 줄이기 위해 데이터 갱신 시점(Python)에 미리 계산해 `data/backtests.json`으로 저장 — 브라우저는 그리기만 함

### 호스팅
- **GitHub Pages 단일 플랫폼** (사이트+데이터+GitHub Actions 자동갱신 모두 한 저장소, 완전 무료, 커스텀 도메인 불필요 — `https://<계정>.github.io/<repo>/` 형태로 공개)
- GitHub Actions가 월 1회(전월 마감 직후) `scripts/update_data.py` 실행 → `data/*.json` 갱신·커밋 → Pages 자동 재배포
- ⚠️ 로컬 환경에 `gh` CLI가 설치되어 있지 않아, 실제 GitHub 저장소 생성·Pages 활성화는 브라이언의 최종 승인/직접 조작이 필요한 단계 (공개 게시 행위이므로 명시적 승인 필요)

## 폴더 구조
| 경로 | 용도 |
|---|---|
| `scripts/update_data.py` | Yahoo/BLS/FRED 등에서 가격·매크로 데이터 수집 → `data/prices.json`, `data/economic.json` |
| `scripts/strategy_engine.py` | 18개 전략 계산 로직의 Python 이식판 (백테스트용) |
| `scripts/build_output.py` | 최신 전월말 기준 신호 계산 + 과거 월별 백테스트 시뮬레이션 → `data/current.json`, `data/backtests.json` |
| `js/engine.js` | 전략 계산 로직 (브라우저용, 참고 사이트 이식 + 전월말 앵커링 수정) |
| `js/app.js` | UI 렌더링, 데이터 로딩, 차트 그리기 |
| `index.html`, `style.css` | 프론트엔드 |
| `.github/workflows/update-data.yml` | 월 1회 자동 데이터 갱신 |
